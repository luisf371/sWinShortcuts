## 2026-07-07 — HoldBreath stuck-Alt deep dive (analysis only, no code changed)
- Root cause of stuck Alt ("Tab acts as Alt+Tab"): in `InputHookService.cs`, hold-breath state transitions are atomic but SendInput side effects run OUTSIDE the atomic region. Primary race: timer thread CAS PENDING→ACTIVE (L963) → hook thread processes RBUTTONUP, CAS ACTIVE→IDLE (L936) + sends Alt-UP (L940) → timer thread then executes SendKey(Alt,true) (L973) LAST → Alt stuck until next full right-click cycle.
- Aggravator: jitter 15-36ms (L894) on top of any delay>0 puts activation at ~20-41ms after button-down — inside typical click duration, so UP races activation constantly. Delay=0 activates synchronously on the hook thread (L915-918) and avoids this race entirely.
